# Never commit — the user commits their own work

The user reviews every change and commits it themselves. Do not run
`git commit`, `git push`, `git tag`, or open pull requests on their behalf, and
do not stage changes unless explicitly asked.

Do not end a response by offering to commit, proposing a commit message, or
listing "next steps" that amount to committing. When a task is done, state what
changed and stop.

The only exception is an explicit, in-the-moment instruction to commit (e.g.
"commit this"). Absent that, leave the working tree for the user to review.
